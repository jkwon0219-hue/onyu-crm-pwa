# 관계 CRM 최신 배포파일

## 업데이트 전 필수
앱에서 먼저 `암호화 백업`을 눌러 백업 파일을 저장하세요.

## 배포 방법
1. 이 zip 파일을 다운로드합니다.
2. 압축을 풉니다.
3. GitHub 저장소 `onyu-crm-pwa`에 들어갑니다.
4. `src/App.jsx`, `src/main.jsx`, `src/styles.css`, `index.html`, `package.json`, `netlify.toml`, `public/manifest.webmanifest`를 업로드/교체합니다.
5. Commit changes를 누릅니다.
6. Netlify가 자동 배포할 때까지 기다립니다.

## 고객 데이터 유지
같은 Netlify 사이트 주소를 계속 쓰면 기존 고객 데이터는 보통 유지됩니다.
고객 데이터는 서버가 아니라 각 기기 브라우저 localStorage에 암호화 저장됩니다.


## 이번 추가사항
- 고객 상세 화면에 `고객 삭제` 버튼 추가
- 삭제 전 확인창 표시
- 삭제 후 다음 고객 자동 선택


## 이번 추가사항
- 고객정보 엑셀 내보내기 버튼 추가
- CSV 파일로 다운로드 가능
- 주민등록번호는 마스킹 처리
- 가족정보, 최근 연락기록, 상담후기, 메모 포함


## 이번 수정사항
- 엑셀 내보내기 시 고객 주민등록번호 전체 표시
- 가족 주민등록번호 전체 표시
- 가족정보를 관계/이름/주민번호/나이/전화번호/메모 칸으로 분리
- 가족 1명당 1줄로 내보내기
- CSV 대신 Excel 호환 .xls 파일 다운로드


## 이번 수정사항
- 엑셀 가져오기 기능 추가
- .xls / .xlsx / .csv 파일 선택 가능
- 고객명/전화번호/주민번호 기준으로 기존 고객 업데이트 및 신규 고객 추가
- 가족정보 열을 읽어 가족 목록으로 적용
- 엑셀 내보내기 파일은 전체 주민번호와 가족정보 분리 형식 유지


## Netlify 배포 실패 수정
이번 버전은 Netlify Functions 번들링 오류를 해결하기 위해 package.json에 아래 의존성을 추가했습니다.

- @netlify/functions
- web-push
- xlsx

기존 netlify/functions 폴더가 저장소에 남아 있어도 배포가 실패하지 않도록 수정한 버전입니다.
